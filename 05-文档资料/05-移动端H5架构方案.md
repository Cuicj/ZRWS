# 智壤卫士 - 移动端 H5 架构方案

## 一、总体架构设计

### 1.1 技术选型

| 层级 | 技术方案 | 说明 |
|------|---------|------|
| 页面技术 | Vue 3 + HTML5 + CSS3 | 单页应用 SPA |
| 样式框架 | Element Plus + 自定义响应式样式 | 移动端自适应 |
| 图表库 | ECharts 5.x | 简化版图表 |
| 地图 | Leaflet + OpenStreetMap/天地图 | 轻量级GIS |
| 原生能力 | Cordova / uni-app WebView 壳 | 套壳APP |
| 数据通信 | HTTPS REST + WebSocket | 与后端微服务交互 |
| 离线存储 | localStorage + IndexedDB | 弱网环境数据缓存 |
| 包管理 | npm | 依赖管理 |

### 1.2 设计原则

```
移动端优先 (Mobile First)
├── 单列布局为主，避免复杂多列
├── 触摸友好：按钮 ≥ 44×44px
├── 减少输入：优先选择/滑动代替键盘输入
├── 数据轻量化：只传输必需字段
├── 流量节省：图片懒加载 + 压缩
└── 断网可用：核心功能离线缓存
```

## 三、页面路由结构（SPA Hash 路由）

```
/ (根路径 → 仪表盘)
├── /login                    登录页
├── /dashboard                运行仪表盘
├── /missions                 任务列表
│   └── /mission/:id          任务详情
│       └── /mission/:id/flight  飞行控制（关联任务）
├── /flight-control           飞行控制中心
├── /gps-track                GPS实时航迹
├── /soil-sample              土壤采样数据
│   └── /soil-sample/new      新建采样记录
├── /area-calc                地块面积测量
├── /disaster-risk            灾害风险评估
├── /approval                 审批中心
│   └── /approval/:id         审批详情
└── /mine                     我的（个人中心）
```

### 3.1 页面跳转策略

- 固定底部 **TabBar**：5 个常驻入口（任务、飞行、采集、审批、我的）
- 其他模块通过 **侧边菜单** 或 **快捷入口** 进入
- 需要专业操作的模块（CAD、3D重建、工作流设计）直接提示"请在PC端操作"

## 四、App 套壳方案（H5 壳）

### 4.1 方案选择：轻量级 WebView 壳

```
用户手机
├── 智壤卫士 APP（20MB 安装包）
│   ├── 系统 WebView / X5 内核
│   ├── 打开远程 H5 地址：https://mobile.zrws.example.com
│   └── 提供原生接口：
│       ├── 摄像头/相册  → 拍照上传
│       ├── GPS 定位     → 获取当前位置
│       ├── 文件系统     → 离线缓存数据
│       ├── 通知推送     → 消息提醒
│       └── 蓝牙/串口    → 连接土壤传感器
└── 优势：一次开发，多端运行，无需频繁更新App
```

### 4.2 APP 更新机制

| 更新类型 | 触发方式 | 用户体验 |
|---------|---------|---------|
| H5 页面更新 | 服务器发布新代码 | 刷新页面即生效，用户无感 |
| 壳 APP 更新 | 应用商店/弹窗提示 | 需要下载安装（仅壳升级时） |
| 原生插件更新 | 随壳 APP 发版 | 需要下载安装 |

## 五、数据交互架构

### 5.1 与后端微服务通信

```
移动端 H5
    │
    │ HTTPS REST API
    │ WebSocket (实时数据流)
    ▼
API Gateway (zrws-gateway)
    │
    ├── zrws-auth        → 认证/登录
    ├── zrws-collect     → 任务/飞行/GPS/采样
    ├── zrws-process     → 数据处理/质检状态
    ├── zrws-analyze     → 分析结果查询
    └── zrws-approval    → 审批/工作流
```

### 5.2 实时数据推送

| 场景 | 技术 | 更新频率 | 数据量 |
|------|------|---------|--------|
| GPS 实时坐标 | WebSocket | 1Hz（每秒1次） | ~100字节 |
| 飞行遥测数据 | WebSocket | 2Hz | ~200字节 |
| 任务审批通知 | WebSocket / 推送 | 事件触发 | 小 |
| 仪表盘数据 | HTTP 轮询 | 30s/次 | ~5KB |

### 5.3 弱网处理策略

1. **数据缓存**：核心数据写入 localStorage，断网时读缓存
2. **请求队列**：离线操作排队，联网后自动提交
3. **渐进式加载**：优先展示文本、延迟加载图片/地图
4. **请求超时**：移动端超时时间 15s（PC端 30s）

## 六、移动端特有功能

### 6.1 调用手机原生能力

| 能力 | 用途 | 模块 |
|------|------|------|
| 手机GPS | 记录采样点经纬度 | 土壤采样 / 面积测量 |
| 摄像头 | 拍照记录现场 | 土壤采样 / 任务执行 |
| 蓝牙 | 连接土壤传感器 | 土壤采样（直接读取pH/水分/EC） |
| 相册 | 上传现场图片 | 各模块通用 |
| 文件系统 | 离线存储航迹/采样数据 | 数据上传 |
| 推送通知 | 任务提醒、审批提醒 | 系统消息 |

### 6.2 移动端交互设计

- **单手操作**：核心按钮放屏幕下半区，便于拇指触碰
- **滑动操作**：列表项左右滑动 = 快捷操作（删除/标记完成）
- **下拉刷新**：列表页下拉刷新最新数据
- **上拉加载**：分页数据无限滚动
- **长按**：长按任务 = 弹出操作菜单（分享/复制/删除）
- **手势缩放**：地图/航迹图支持双指缩放

## 七、性能优化要点

| 优化点 | 技术方案 | 目标 |
|--------|---------|------|
| 首屏加载 | 代码分割 + 懒加载 | ≤ 3s |
| 静态资源 | CDN 分发 + Gzip 压缩 | JS 包 ≤ 500KB |
| 图片 | WebP 格式 + 懒加载 + 占位 | 单图 ≤ 100KB |
| 数据请求 | 请求合并 + 缓存策略 | 关键接口 ≤ 500ms |
| 内存管理 | 切换页面时销毁图表/地图实例 | OOM < 1% |
| 流量控制 | WiFi 下同步完整数据，4G 下仅同步摘要 | 日均 ≤ 50MB |

## 九、与 PC 端的差异

| 维度 | PC 端（Web） | 移动端（H5 + APP壳） |
|------|-------------|---------------------|
| 屏幕 | ≥ 1280px | 320px ~ 768px |
| 操作方式 | 鼠标 + 键盘 | 触摸 + 手势 |
| 数据展示 | 完整数据表格 | 摘要卡片 + 关键字段 |
| 数据录入 | 完整表单 | 简化表单 + 扫码/拍照 |
| GIS/图表 | 完整交互 + 3D | 2D 简化版 |
| CAD/3D重建 | 完整功能 | 仅查看摘要 + 跳转提示 |
| 审批流配置 | 完整 BPMN 设计器 | 仅查看/操作待办 |
