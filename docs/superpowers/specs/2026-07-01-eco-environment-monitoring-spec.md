# 生态环境监测模块设计规格说明书

> **版本**: v1.0 | **日期**: 2026-07-01 | **状态**: 待评审

---

## 1. 概述

### 1.1 项目背景

智壤卫士系统已具备土地资源管理、灾害风险评估等功能。为完善生态环境监测能力，新增气候变暖监测、沙漠化监测功能，并增强水土流失监测能力，形成完整的生态环境监测体系。

### 1.2 设计目标

- **部分扩展**：增强现有灾害风险模块中的水土流失监测能力
- **部分新增**：新增气候变暖监测、沙漠化监测两个独立功能模块
- **数据先行**：建立完整的生态环境标准数据体系，包括国际标准、国家标准、安全阈值、基础分区数据
- **统一架构**：与现有系统保持一致的技术栈和代码风格

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **数据驱动** | 先有标准数据，再有业务功能，确保功能有数据支撑 |
| **增量开发** | 不破坏现有功能，通过新增表、新增页面的方式扩展 |
| **标准化** | 所有指标、阈值、分级均基于国际或国家标准 |
| **可扩展** | 预留扩展字段，方便后续接入更多生态监测类型 |
| **一致性** | 与现有灾害风险模块保持一致的代码风格和交互模式 |

| `reference_standard` | VARCHAR(255) | 否 | 参考标准文号 |
| `sort_order` | INT | 否 | 排序，默认0 |
| `status` | VARCHAR(16) | 否 | 状态：ACTIVE-启用, INACTIVE-停用 |

### 3.4 水土流失监测（扩展现有灾害风险表）

**方案**：不新增物理表，复用现有 `zrws_disaster_risk` 表，通过以下方式扩展：

1. **disasterType = SOIL_EROSION** 标识水土流失类型
2. **monitoringData JSON 字段** 存储详细的水土流失指标：
   - `erosionType` - 侵蚀类型：HYDRAULIC-水力, WIND-风力, FREEZE-冻融, GRAVITY-重力
   - `erosionIntensity` - 侵蚀强度等级
   - `erosionModulus` - 侵蚀模数(t/km²·a)
   - `soilLossAmount` - 土壤流失量(t)
   - `usleFactors` - USLE各因子值(R, K, L, S, C, P)
   - `soilLossTolerance` - 土壤容许流失量

### 7.3 水土流失风险评估（USLE模型）

**公式**：`A = R × K × L × S × C × P`

| 因子 | 含义 | 说明 |
|------|------|------|
| R | 降雨侵蚀力因子 | 降雨强度和降雨量 |
| K | 土壤可蚀性因子 | 土壤抗侵蚀能力 |
| L | 坡长因子 | 地形因素 |
| S | 坡度因子 | 地形因素 |
| C | 植被覆盖因子 | 植被保护作用 |
| P | 水土保持措施因子 | 人为干预 |
