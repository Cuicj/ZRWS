## 1. 项目概述

**智壤卫士 (Smart Soil Guardian)** 是一套面向土地资源管理的智能审批与监测系统，集成了工作流引擎、GIS地理信息、AI智能分析等功能。

- **项目名称**：智壤卫士 - 审批服务
- **版本号**：2.0.0
- **核心功能**：
  - 📋 审批工作流管理（Flowable 7.0.0）
  - 🌍 3D GIS地理信息展示（Cesium）
  - 🤖 AI智能数据分析（Spring AI）
  - 📰 土地资源公告栏（AI分析时间线）
  - 📊 土地资源数据可视化
  - 🛰️ 无人机采集任务管理
  - 🏗️ BPMN流程设计器

## 5. 快速部署（一键脚本）

### 5.1 Linux一键部署

项目提供了自动化部署脚本 `deploy/deploy.sh`，可用于快速部署。

```bash
# 1. 克隆或上传代码到服务器
git clone <仓库地址> /app/ZRWS
cd /app/ZRWS

# 2. 配置部署脚本
vim deploy/deploy.sh
# 修改以下配置：
# - APP_DIR: 应用部署目录
# - REPO_DIR: Git仓库目录
# - BACKEND_DIR: 后端代码目录

# 3. 赋予执行权限
chmod +x deploy/deploy.sh

# 4. 执行部署
./deploy/deploy.sh
```

### 5.2 部署脚本功能说明

`deploy.sh` 脚本自动完成以下操作：

1. ✅ 拉取最新代码
2. ✅ Maven编译后端
3. ✅ 停止旧服务
4. ✅ 备份旧版本
5. ✅ 部署新版本
6. ✅ 启动服务
7. ✅ 健康检查
8. ✅ 自动清理旧备份（保留最近5个）

### 5.3 部署前检查清单

部署前请确认以下事项：

- [ ] Java 17+ 已安装并配置环境变量
- [ ] Maven 3.8+ 已安装
- [ ] MySQL 8.0+ 服务已启动
- [ ] Redis 6.0+ 服务已启动
- [ ] Nginx 已安装
- [ ] 数据库 `zrws_approval` 已创建
- [ ] 数据库用户权限已配置
- [ ] 防火墙已开放 80/443/5571 端口

## 6. Linux环境部署

### 6.1 后端部署

#### 6.1.1 源码编译打包

```bash
# 进入后端目录
cd code/java

# 安装公共模块
mvn install -pl zrws-common -am -DskipTests

# 打包审批服务
mvn clean package -pl zrws-approval -am -DskipTests

# 打包完成后，jar包位置
# zrws-approval/target/zrws-approval.jar
```

#### 6.1.2 部署目录结构

```bash
# 创建部署目录
mkdir -p /app/{logs,config,uploads,backups}

# 复制jar包
cp zrws-approval.jar /app/app.jar

# 复制配置文件（可选，外部化配置）
cp application.yml /app/config/
```

#### 6.1.3 配置说明

主要配置项（`application.yml`）：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `server.port` | 服务端口 | 5571 |
| `server.servlet.context-path` | 上下文路径 | /approval |
| `spring.datasource.url` | 数据库连接 | - |
| `spring.data.redis.host` | Redis地址 | localhost |
| `flowable.database-schema-update` | Flowable自动建表 | true |
| `flowable.rest-api.enabled` | Flowable REST API | false |
| `zrws.dba.auto-sync-on-startup` | DBA自动同步表结构 | true |

#### 6.1.4 环境变量配置

```bash
# 生产环境建议使用环境变量管理敏感信息
export SPRING_DATASOURCE_URL="jdbc:mysql://localhost:3306/zrws_approval?useSSL=true&serverTimezone=Asia/Shanghai"
export SPRING_DATASOURCE_USERNAME="your_username"
export SPRING_DATASOURCE_PASSWORD="your_password"
export REDIS_HOST="localhost"
export REDIS_PORT="6379"
export REDIS_PASSWORD="your_redis_password"
export OPENAI_API_KEY="your_api_key"
export OPENAI_BASE_URL="https://api.openai.com"
```

#### 6.1.5 启动服务

```bash
# 直接启动（测试用）
java -jar /app/app.jar

# 指定配置文件
java -jar /app/app.jar --spring.config.location=/app/config/application.yml

# 指定JVM参数
java -Xms512m -Xmx1024m -XX:+UseG1GC -jar /app/app.jar

# 后台运行
nohup java -jar /app/app.jar > /app/logs/app.log 2>&1 &
```

#### 6.1.6 验证启动

```bash

netstat -tlnp | grep 5571

# 健康检查

# DBA同步状态检查
curl http://localhost:5571/approval/api/v1/dba/status

# 查看日志
tail -f /app/logs/app.log
```

### 6.3 Nginx配置

#### 6.3.1 安装Nginx

```bash
# CentOS
yum install -y nginx

# Ubuntu
apt install -y nginx
```

#### 6.3.2 配置文件

部署配置文件：[`deploy/nginx.conf`](deploy/nginx.conf)

```bash
# 复制配置
cp deploy/nginx.conf /etc/nginx/conf.d/zrws.conf

# 修改配置中的域名
sed -i 's/zrws.example.com/your-domain.com/g' /etc/nginx/conf.d/zrws.conf

# 检查配置
nginx -t

# 重载配置
nginx -s reload
```

#### 6.3.3 关键配置说明

**反向代理后端API：**

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:5571/approval/api/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_connect_timeout 60s;
    proxy_send_timeout 60s;
    proxy_read_timeout 60s;
    client_max_body_size 50m;
}
```

> ⚠️ **重要**：`proxy_pass` 路径必须包含 `/approval/api/`，与后端 `context-path` 对应，否则会出现405错误。

**前端静态资源：**

```nginx
location / {
    root /app/frontend/dist;
    index index.html;
    try_files $uri $uri/ /index.html;
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 6.3.4 SSL证书配置

```bash
# 创建证书目录
mkdir -p /etc/nginx/ssl

# 复制证书文件
cp your_domain.crt /etc/nginx/ssl/zrws.crt
```

#### 7.3.3 web.config配置

在前端目录创建 `web.config` 文件：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="API Proxy" stopProcessing="true">
          <match url="^api/(.*)" />
          <action type="Rewrite" url="http://127.0.0.1:5571/approval/api/{R:1}" />
        </rule>
        <rule name="Vue History Mode" stopProcessing="true">
          <match url=".*" />
          <conditions logicalGrouping="MatchAll">
            <add input="{REQUEST_FILENAME}" matchType="IsFile" negate="true" />
            <add input="{REQUEST_FILENAME}" matchType="IsDirectory" negate="true" />
          </conditions>
          <action type="Rewrite" url="/index.html" />
        </rule>
      </rules>
    </rewrite>
    <staticContent>
      <mimeMap fileExtension=".woff" mimeType="application/font-woff" />
      <mimeMap fileExtension=".woff2" mimeType="application/font-woff2" />
    </staticContent>
  </system.webServer>
</configuration>
```

## 8. Docker容器化部署

### 8.1 Docker Compose部署

#### 8.1.1 创建docker-compose.yml

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: zrws-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root_password
      MYSQL_DATABASE: zrws_approval
      MYSQL_USER: zrws_user
      MYSQL_PASSWORD: zrws_password
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./code/sql:/docker-entrypoint-initdb.d
    command:
      - --character-set-server=utf8mb4
      - --collation-server=utf8mb4_unicode_ci

  redis:
    image: redis:6-alpine
    container_name: zrws-redis
    restart: always
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    command: redis-server --appendonly yes

  backend:
    build:
      context: ./code/java
      dockerfile: Dockerfile
    container_name: zrws-backend
    restart: always
    ports:
      - "5571:5571"
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/zrws_approval?useSSL=false&serverTimezone=Asia/Shanghai
      - SPRING_DATASOURCE_USERNAME=zrws_user
      - SPRING_DATASOURCE_PASSWORD=zrws_password
      - SPRING_DATA_REDIS_HOST=redis
      - SPRING_DATA_REDIS_PORT=6379
      - OPENAI_API_KEY=your-api-key
      - JAVA_OPTS=-Xms512m -Xmx1024m
    depends_on:
      - mysql
      - redis
    volumes:
      - ./logs:/app/logs
      - ./uploads:/app/uploads

  frontend:
    build:
      context: ./code/html
      dockerfile: Dockerfile
    container_name: zrws-frontend
    restart: always
    ports:
      - "80:80"
      - "443:443"
    depends_on:
      - backend
    volumes:
      - ./deploy/nginx.conf:/etc/nginx/conf.d/default.conf
      - ./ssl:/etc/nginx/ssl

volumes:
  mysql_data:
  redis_data:
```

#### 8.1.2 启动服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 查看服务状态
docker-compose ps

# 停止服务
docker-compose down
```

### 9.2 自动建表机制

系统启动时会自动执行以下操作：

1. **Flowable引擎表**：Flowable自动创建（`ACT_*` 开头的表）
   - 配置项：`flowable.database-schema-update: true`

2. **业务表自动同步**：DBA自动扫描实体类，同步表结构
   - 配置项：`zrws.dba.auto-sync-on-startup: true`
   - 表前缀：`zrws_`
   - 实体类扫描包：`com.zrws.approval.domain.entity`
   - 排除框架表前缀：`ACT_`, `FLW_`

3. **菜单数据初始化**：首次启动自动初始化菜单
   - 初始化类：`MenuDataInitializer`
   - 初始化脚本：`sql/sys_menu_init.sql`

### 9.3 DBA Schema同步说明

DBA自动同步功能会在以下时机执行：

- 上下文刷新时（`ContextRefreshedEvent`）
- 应用启动完成后（`ApplicationReadyEvent`）
- 手动调用API：`POST /api/v1/dba/sync`

同步功能特性：

- ✅ 自动扫描实体类，创建缺失的表
- ✅ 自动添加缺失的列
- ✅ 驼峰命名自动转换为下划线命名
- ✅ 支持Mapper XML中自定义表/字段扫描
- ✅ 排除Flowable等框架表
- ✅ 详细日志输出
- ✅ 同步失败不中断启动（可配置）

## 16. 故障排查手册

### 16.1 服务无法启动

**检查步骤：**

```bash
# 1. 检查服务状态
systemctl status zrws

# 2. 查看系统日志
journalctl -u zrws -n 100

# 3. 查看应用日志
tail -n 100 /app/logs/zrws-error.log

# 4. 检查端口占用
netstat -tlnp | grep 5571

# 5. 检查Java进程
ps aux | grep java

# 6. 手动启动测试
java -jar /app/app.jar
```

**常见原因：**

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 端口被占用 | 其他进程占用5571端口 | `lsof -i:5571` 查找并结束进程 |
| 数据库连接失败 | 数据库未启动或配置错误 | 检查数据库服务和连接配置 |
| 内存不足 | JVM堆内存设置过大 | 调整 `-Xmx` 参数 |
| 配置文件错误 | yml格式错误 | 检查 `application.yml` 语法 |

### 16.2 数据库连接失败

**错误信息：** `Cannot create PoolableConnectionFactory`

**排查步骤：**

```bash
# 1. 检查MySQL服务状态
systemctl status mysqld

# 2. 测试数据库连接
mysql -h localhost -u zrws_user -p zrws_approval

# 3. 检查数据库是否存在
mysql -u root -p -e "SHOW DATABASES LIKE 'zrws_approval';"

# 4. 检查用户权限
mysql -u root -p -e "SHOW GRANTS FOR 'zrws_user'@'%';"

# 5. 检查防火墙
firewall-cmd --list-ports
```

### 16.3 Flowable启动报错

**错误信息：** `Tables not found` 或 `couldn't check for upgradable tables`

**排查步骤：**

```bash
# 1. 确认Flowable配置
grep "flowable" application.yml

# 2. 检查数据库中是否有ACT_表
mysql -u root -p zrws_approval -e "SHOW TABLES LIKE 'ACT_%';"

# 3. 确认配置正确
# flowable.database-schema-update: true
# flowable.rest-api.enabled: false
```

**解决方案：**

确保 `application.yml` 中配置：

```yaml
flowable:
  database-schema-update: true
  rest-api:
    enabled: false
```

### 16.4 前端页面404

**排查步骤：**

1. 检查Nginx配置是否正确
2. 确认静态文件路径是否存在
3. 检查 `try_files` 配置（支持Vue Router history模式）
4. 检查文件权限

```bash
# 检查Nginx配置
nginx -t

# 检查静态文件
ls -la /app/frontend/

# 检查Nginx错误日志
tail -f /var/log/nginx/zrws-error.log
```

### 16.5 API请求返回405

**错误信息：** `405 Method Not Allowed`

**常见原因：**

1. Nginx代理配置不正确，POST请求被拦截
2. 前端请求路径与后端不匹配
3. Flowable REST API与自定义路径冲突

**解决方案：**

1. 确保Nginx配置中 `/api/` 代理正确：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:5571/approval/api/;
}
```

2. 禁用Flowable REST API：

```yaml
flowable:
  rest-api:
    enabled: false
```

3. 直接测试后端接口：

```bash
curl -X POST http://localhost:5571/approval/api/v1/designer/drafts \
  -H "Content-Type: application/json" \
  -d '{}'
```

### 16.6 DBA自动同步不生效

**排查步骤：**

```bash
# 1. 检查应用日志中是否有DBA相关日志
grep "\[DBA\]" /app/logs/zrws.log

# 2. 调用DBA状态API
curl http://localhost:5571/approval/api/v1/dba/status

# 3. 手动触发同步
curl -X POST http://localhost:5571/approval/api/v1/dba/sync

# 4. 检查配置
grep -A 10 "zrws.dba" application.yml
```

**常见原因：**

| 原因 | 解决方法 |
|------|----------|
| 实体类扫描包配置错误 | 检查 `entity-package` 配置 |
| 自动同步未启用 | 确认 `auto-sync-on-startup: true` |
| DBA功能未启用 | 确认 `enabled: true` |
| 表已存在但字段不匹配 | 检查实体类字段名与数据库列名 |

### 16.7 Cesium加载失败

**排查步骤：**

1. 确认 `vite-plugin-cesium` 已正确配置
2. 检查Cesium静态资源路径
3. 查看浏览器控制台错误信息

**解决方案：**

确保 `vite.config.js` 中配置：

```javascript
import cesium from 'vite-plugin-cesium';

export default defineConfig({
  plugins: [vue(), cesium()],
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          cesium: ['cesium']
        }
      }
    }
  }
});
```

### 16.8 内存不足/OOM

**排查步骤：**

```bash
# 1. 查看内存使用

# 3. 查看GC日志
tail -f /app/logs/gc.log

# 4. 检查是否有堆转储文件
ls -la /app/logs/*.hprof
```

**解决方案：**

```bash
# 调整JVM参数
java -Xms2048m -Xmx2048m -XX:+UseG1GC -jar app.jar

# 分析堆转储
jhat /app/logs/heapdump.hprof
```

## 18. 常见问题FAQ

### Q1: 启动报错 "Cannot create PoolableConnectionFactory"

**原因**：数据库连接失败

**解决**：
1. 检查数据库服务是否启动
2. 验证连接地址、用户名、密码
3. 确认数据库已创建

```bash
# 测试数据库连接
mysql -h localhost -u zrws_user -p zrws_approval
```

### Q2: Flowable启动报错 "Tables not found"

**原因**：Flowable表未创建

**解决**：
确认配置 `flowable.database-schema-update: true`，首次启动会自动建表。

### Q3: 前端页面404

**原因**：Nginx配置不正确或静态文件路径错误

**解决**：
1. 检查 `root` 路径是否正确
2. 确认 `try_files` 配置正确（支持Vue Router history模式）
3. 检查文件权限

### Q4: API请求返回405

**原因**：Nginx代理配置不正确，POST请求被拦截

**解决**：
确保Nginx配置中：
```nginx
location /api/ {
    proxy_pass http://127.0.0.1:5571/approval/api/;
    # ... 其他配置
}
```

> ⚠️ 注意：`proxy_pass` 必须包含 `/approval/api/`，否则路径不匹配

同时确保Flowable REST API已禁用：
```yaml
flowable:
  rest-api:
    enabled: false
```

### Q5: Cesium加载失败

**原因**：Cesium静态资源路径问题

**解决**：
确认 `vite-plugin-cesium` 已正确配置，构建时会自动处理Cesium静态资源。

### Q6: DBA自动同步不生效

**原因**：实体类扫描路径配置错误

**解决**：
检查配置：
```yaml
zrws:
  dba:
    enabled: true
    auto-sync-on-startup: true
    entity-package: com.zrws.approval.domain.entity
```

### Q7: 内存不足导致服务OOM

**原因**：JVM堆内存设置过小

**解决**：
调整JVM参数：
```bash
java -Xms1024m -Xmx2048m -XX:+UseG1GC -jar /app/app.jar
```

### Q8: 数据库表没有 zrws_ 前缀

**原因**：DBA同步未执行或实体类未正确配置表名

**解决**：
1. 确认实体类使用 `@TableName("zrws_xxx")` 注解
2. 启用DBA自动同步
3. 手动调用DBA同步API：
```bash
curl -X POST http://localhost:5571/approval/api/v1/dba/sync
```

### Q9: bpmn.js流程设计器保存失败

**原因**：可能是Flowable REST API冲突或路径不匹配

**解决**：
1. 禁用Flowable REST API
2. 检查Nginx代理配置
3. 直接测试后端接口：
```bash
curl -X POST http://localhost:5571/approval/api/v1/designer/drafts \
  -H "Content-Type: application/json" \
  -d '{"name":"test"}'
```

### Q10: 菜单数据未加载

**原因**：菜单初始化失败或表不存在

**解决**：
1. 检查 `zrws_sys_menu` 表是否存在
2. 检查 `MenuDataInitializer` 是否执行
3. 手动执行初始化脚本：
```bash
mysql -u root -p zrws_approval < code/java/zrws-approval/src/main/resources/sql/sys_menu_init.sql
```
